# AGENTS.md

## Repository Overview

Project-Real Alpha is a private Next.js App Router application for an authenticated personal "Alpha Command" launchpad. The current product surface centers on a Rise & Grind dashboard where signed-in users record wake time, sleep duration, mood, date, and gratitude entries.

Primary stack:

| Area | Technology |
| --- | --- |
| App framework | Next.js `16.2.3` with App Router under `src/app` |
| UI runtime | React `19.2.4`, React DOM `19.2.4` |
| Language | TypeScript with `strict: true` |
| Styling | Tailwind CSS v4 via `@tailwindcss/postcss`, global tokens in `src/app/globals.css` |
| Auth | Clerk via `@clerk/nextjs` |
| Data | MongoDB Node driver |
| Package manager | npm, with `package-lock.json` committed |
| Linting | ESLint 9 with `eslint-config-next` core web vitals and TypeScript presets |

Important runtime assumptions:

- This is Next.js 16, not older Next.js. Before changing framework conventions, routing, Proxy, route handlers, caching, or config, read the relevant local guide in `node_modules/next/dist/docs/`.
- Next.js 16 requires Node.js `20.9.0+`; this repo does not currently include `.nvmrc`, `.node-version`, or `engines`.
- `next dev` and `next build` use Turbopack by default in Next.js 16.
- Clerk and MongoDB environment variables are required for authenticated/data-backed flows.

## Code Style And Conventions

- TypeScript is strict; avoid `any` and prefer typed payloads, unions, and narrow parsing for external input.
- Use the `@/*` path alias for imports from `src`.
- Server route handlers should return `Response.json(...)` and avoid leaking internal errors to clients.
- Validate all request payloads before writing to MongoDB. The Rise & Grind endpoint uses `parseRiseAndGrindInput`; keep validation logic close to the domain model in `src/lib`.
- Use Clerk `auth()` in server code before accessing user-scoped data.
- Use `"use client"` only for components that need client hooks, browser APIs, or event handlers.
- Keep React components focused. Shared UI belongs in `src/components`; domain parsing and persistence helpers belong in `src/lib` or route handlers.
- Prefer existing CSS/token patterns in `globals.css`: `surface`, `surface-low`, `surface-high`, `primary`, `font-display`, and compact radii.
- The current visual direction is documented in `DESIGN.md`: dark "Kinetic Vault" surfaces, electric blue accents, sharp/compact geometry, and minimal borders.
- Comments should explain non-obvious decisions, not restate code.

## Architecture Guidance

Layer boundaries:

| Layer | Belongs in | Guidance |
| --- | --- | --- |
| Routes/layouts | `src/app/**` | Keep App Router file conventions intact; use route groups for organization |
| Client UI | `src/components/**` | Add `"use client"` only when required; keep data mutations explicit |
| API handlers | `src/app/api/**/route.ts` | Authenticate, validate, call MongoDB, return stable JSON |
| Domain validation | `src/lib/*-model.ts` | Keep reusable parsing/type logic independent from React |
| Data access | `src/lib/mongodb.ts` | Reuse the MongoDB singleton; do not create ad hoc clients in handlers |
| Auth boundary | `src/proxy.ts`, Clerk APIs | Keep Proxy lightweight; do not do slow data fetching in Proxy |
| Design system | `DESIGN.md`, `src/app/globals.css` | Extend existing tokens before adding one-off palettes |

Next.js-specific rules:

- This repo uses `src/proxy.ts`, the Next 16 replacement convention for Middleware. Do not add `middleware.ts`.
- Only one Proxy file is supported; factor helper logic into modules if Proxy grows.
- For dynamic route handler params in Next 16, follow the local docs: `params` is a promise in route handler context.
- Check `node_modules/next/dist/docs/01-app/` before using framework features that may have changed.

MongoDB/data rules:

- User-owned records must include and query by Clerk `userId`.
- Existing document types are `riseandgrind` and `lead` in a shared collection.
- `riseandgrind` entries are upserted by `{ userId, type, date }` and older duplicates for the same date are removed.
- Be careful when changing field names such as `wakeUpTime`, `sleepDurationHours`, `mood`, `state`, `gratitude`, `date`, `createdAt`, and `updatedAt`; existing data may depend on them.

## Troubleshooting

| Symptom | Likely cause | Suggested action |
| --- | --- | --- |
| `Missing MONGODB_URI environment variable.` | `.env.local` is missing MongoDB config | Add local env values; do not commit them |
| API returns `401 Unauthorized` | User is not signed in or Clerk env is missing | Verify Clerk keys and sign-in flow |
| `/api/health` returns database disconnected | MongoDB URI, network, database permissions, or server availability issue | Check `MONGODB_URI`, database access, and MongoDB status |
| Build/type errors around Next conventions | Next 16 API or file convention mismatch | Read the relevant local doc in `node_modules/next/dist/docs/01-app/` |
| Lint warning for `utilityItems` | Unused constant in command nav | Remove/use it only when working in that area |
| Auth redirects do not behave as expected | Clerk route/proxy/client redirect mismatch | Inspect `src/proxy.ts`, sign-in/sign-up pages, and client `useAuth()` logic |
